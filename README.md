# Система детекции занятости столиков

> Прототип CV-пайплайна для аналитики ресторанного зала: кто сидит, когда ушёл, как быстро пришёл следующий.

---

## Описание проекта

Система анализирует видеопоток с камеры и отслеживает один выбранный столик.

Решаемые задачи:

- Определить момент, когда столик освобождается
- Зафиксировать момент подхода следующего человека
- Вычислить задержку между этими событиями

Применение: аналитика кафе и ресторанов — скорость уборки, поток клиентов, загруженность зала.

## Установка и запуск

**Установить зависимости:**

```bash
pip install -r requirements.txt
```

**Запустить:**

```bash
python main.py --video video.mp4
```

После запуска откроется окно — выберитеgit add . зону столика мышью и нажми `Enter`.

## Используемые данные

Тестирование проводилось на видео: `video2.mp4`.

Зона интереса (ROI) выбирается вручную через `cv2.selectROI` при каждом запуске. В видео отслеживался **столик в правом нижнем углу кадра**.

## Логика решения

### Детекция объектов

Основной метод — **YOLOv8n (Ultralytics)**, детекция только класса `person`.  
При отсутствии модели автоматически включается fallback: **детекция движения (MOG2)**.

### Состояния столика

| Состояние | Цвет рамки | Условие |
|-----------|-----------|---------|
| `EMPTY` — Пусто | 🟢 Зелёный | Людей в ROI нет |
| `OCCUPIED` — Занято | 🔴 Красный | Хотя бы один bounding box пересекает ROI |
| `APPROACHING` — Подход | 🟠 Оранжевый | Переход `EMPTY → OCCUPIED` |

### Стабилизация (Debounce)

Состояние меняется только если оно удерживается **не менее 15 кадров** подряд — это исключает ложные срабатывания от случайных ошибок детекции.

### Расчёт метрики

Для каждой пары событий `EMPTY → APPROACHING` фиксируется задержка.  
По всем интервалам считается минимум, максимум и **среднее время ожидания**.

## Результаты

После обработки формируются три файла:

| Файл | Содержимое |
|------|-----------|
| `output.mp4` | Видео с визуализацией: рамка ROI, статус, тайм-код |
| `events.csv` | Структурированный лог всех событий |
| `report.txt` | Текстовый отчёт со статистикой |

Ключевая метрика — **среднее время между уходом посетителя и подходом следующего** — рассчитывается автоматически и выводится в `report.txt`.

## Ограничения и проблемный кейс

![Проблемный кадр](screenshots/problem_frame.png)

В одном из кадров система определяет столик как `EMPTY`, хотя фактически он занят.

**Причина:** голова человека частично перекрыта лампой (окклюзия) — YOLO не детектирует человека, bounding box отсутствует, система ошибочно фиксирует стол как пустой.
